# 合成引擎 Synthesis Engine
## 推理框架 + 洞察生成 + 冲突解决协议

> 数据是原料，洞察是产品。合成引擎将53个ARU采集的信息
> 转化为结构性判断，这是区分"资料汇编"与"真正研报"的核心差距。

---

## 一、合成的层次模型

```
Level 4: 战略洞察 ← 极少人能做到（本引擎目标）
    ↑ 为什么重要？对谁重要？现在有什么行动机会？
    
Level 3: 结构性判断
    ↑ 这些现象说明了什么？行业逻辑是什么？
    
Level 2: 关联与模式
    ↑ 数据点之间有什么联系？有没有矛盾？
    
Level 1: 事实陈述 ← 大多数AI报告止步于此
    ↑ XX市场规模XXX亿元，同比增长XX%
```

**强制要求**：每份报告的洞察密度 Level 3+4 ≥ 40%

---

## 二、核心推理框架

### 框架一：五力穿透法（超越波特）

标准五力分析往往流于表面。执行以下升级版本：

```
原始问题（表层）        →  穿透问题（深层）
─────────────────────────────────────────────────
供应商议价能力高       →  为什么？是因为技术壁垒还是地理集中？
                          未来5年这个力量方向会变吗？
                          哪家公司正在绕过这个力量？

购买者议价能力高       →  哪个细分客群的议价力最弱？（这是机会）
                          价格敏感度来自什么？信息不对称还是转换成本？

替代品威胁           →  替代是全量替代还是部分功能替代？
                          替代需要多大的行为改变成本？谁是潜在推手？

新进入者威胁         →  谁是最危险的潜在进入者（有能力+有动机）？
                          现有壁垒是永久性的还是会被技术侵蚀？

行业内竞争           →  竞争维度是什么？（价格/技术/渠道/品牌）
                          竞争烈度的变化方向？触发战争的引爆点是什么？
```

---

### 框架二：利润池归属法

```
步骤1: 标出价值链所有环节
步骤2: 估算每个环节的收入规模
步骤3: 估算每个环节的利润率
步骤4: 计算各环节利润池绝对值
步骤5: 标出利润池的流动方向（过去5年发生了什么转移？）
步骤6: 判断未来利润池会流向哪里，为什么

关键洞察生成点：
→ "利润池正在从XX环节流向XX环节，原因是XX"
→ "XX环节拥有不成比例的利润占比，因为XX"
→ "XX是利润黑洞，所有玩家都在亏损，但没人能退出，因为XX"
```

---

### 框架三：变化速率差分析

```
维度A（变化快）vs 维度B（变化慢）→ 产生结构性机会/风险

例：
技术变化速度（快）vs 监管调整速度（慢）
→ 监管套利窗口，先进者优势

消费者需求变化（快）vs 供给侧调整（慢）
→ 供需缺口，定价权向供给侧集中

行业整体增长（慢）vs 细分赛道增长（快）
→ 份额零和博弈，关注赢家通吃趋势
```

---

### 框架四：反向归因法（Anti-Consensus思维）

> 世界顶级研报的标志：说出大多数人没想到的话，并且是对的。

```
步骤1: 列出行业内的"市场共识"（5-7条）
步骤2: 对每条共识问：
       "这个共识基于什么假设？"
       "这个假设在什么条件下会失效？"
       "如果假设失效，结论如何反转？"
步骤3: 识别1-2个最脆弱的共识
步骤4: 构建反向逻辑链，寻找支撑反向的证据

输出格式：
"市场共识：XXX
我们认为：XXX
核心分歧：XXX
支持证据：XXX
需要监测的信号：XXX"
```

---

### 框架五：时间尺度叠加法

```
将行业动态分解为三个时间层：

短期（0-18个月）：周期性波动
→ 库存周期、季节效应、政策脉冲、宏观摆动
→ 这些会掩盖还是放大真实信号？

中期（2-5年）：结构性变化
→ 新技术采用曲线、监管框架成型、竞争格局重塑
→ 谁是这个时间框架的赢家？

长期（5年以上）：范式转换
→ 需求的根本性变化、替代性技术成熟、人口结构
→ 当前格局是否可持续？

关键动作：当短期和长期信号矛盾时，清楚标注并解释
```

## 三、数据冲突解决协议

当不同来源数据矛盾时，执行以下决策树：

```
冲突检测
    ↓
差距 < 10%？
├─ 是 → 取均值，脚注说明差距
└─ 否 ↓

差距 10-30%？
├─ 检查：口径是否一致（定义/地理/时间/货币）？
│   ├─ 口径不同 → 统一口径后再比较
│   └─ 口径相同 → 分析差距来源，保留两组数据
└─ 保留最权威来源为主，其他为参考，注明差距

差距 > 30%？
├─ 标注为"高不确定性数据"
├─ 呈现所有数据点及其来源
├─ 分析造成巨大差距的可能原因
└─ 避免以此数据支撑核心结论

[DATA GAP] 处理：
├─ 明确标注无法找到可靠数据
├─ 说明已尝试的搜索路径
├─ 提供数量级估算（注明是估算+方法）
└─ 建议获取此数据的其他途径
```

## 四、洞察生成协议

每个主要发现需通过以下三个问题提升至洞察：

```
问题1: "So What?" ← 这对谁意味着什么？
问题2: "Why Now?" ← 为什么这个时间点特别重要？
问题3: "What Next?" ← 接下来最可能发生什么？
```

**洞察质量分级**：

| 级别 | 描述 | 示例 |
|------|------|------|
| **Insight-A** | 反直觉 + 有数据 + 可操作 | "X行业看似竞争激烈，但前三名的毛利率在过去3年逆势提升5pp，因为它们都完成了从产品到平台的转型，而其余玩家仍在产品层竞争" |
| **Insight-B** | 非显而易见 + 有数据支撑 | "Y行业中，市场份额≥15%的玩家毛利率均值比＜5%的玩家高出23pp，规模效应在此行业极其显著" |
| **Insight-C** | 有数据的描述性结论 | "X行业市场规模2024年达XXX亿元，增速12%" |
| **描述性陈述** | 无数据的泛化描述 | "X行业竞争激烈，市场前景广阔" ← 严禁出现 |

**要求**：研报中 Insight-A ≥ 3个，Insight-B ≥ 8个

## 五、叙事架构（Narrative Architecture）

世界级研报的叙事不是"信息罗列"，而是"有逻辑的论证"。

### 5.1 研报叙事的三种结构

**结构一：假设-检验型**（适合投资研究）
```
开篇论点 → 证据1（支持）→ 证据2（支持）→ 反驳意见 → 
回应反驳 → 情景分析 → 结论强化
```

**结构二：现象-机制-预测型**（适合战略研究）
```
现象描述 → 为什么会这样（机制）→ 这个机制下未来会怎样 → 
谁将受益/受损 → 应对建议
```

**结构三：问题-选项-判断型**（适合决策支持）
```
核心决策问题 → 行业现实约束 → 可行选项A/B/C → 
各选项优劣分析 → 推荐判断 + 执行条件
```

### 5.2 段落级质量标准

每个段落必须满足：
- 首句是结论（不是铺垫）
- 中间句是证据/机制
- 尾句是推进（连接下一段或升华）

**禁止出现的句式**：
```
❌ "随着经济的发展，XX行业也迎来了新的机遇"
❌ "综上所述，XX行业具有良好的发展前景"
❌ "XX行业竞争激烈，企业需要不断提升竞争力"
❌ "未来，XX行业将持续保持稳定增长态势"
```

## 六、情景分析协议

对于影响重大的不确定因素，必须构建情景矩阵：

```
步骤1: 识别2个最关键的不确定性轴（非可预测的、影响大的）
       例：监管严格程度 × 技术突破速度

步骤2: 构建2×2矩阵，命名四个情景
       情景名称应该有画面感，不能叫"情景A/B/C/D"

步骤3: 对每个情景：
       - 描述世界如何演化到这里（2-3句叙事）
       - 行业规模/格局在此情景下的定量变化
       - 谁是赢家，谁是输家
       - 监测信号（如何判断我们正在走向这个情景）

步骤4: 为每个情景赋予主观概率（四者之和=100%）
步骤5: 基础情景下给出核心建议，并说明在其他情景下如何调整
```

## 七、自我审查清单（合成完成后执行）

```
逻辑质量：
□ 每个核心结论都有2个以上独立来源支撑
□ 因果链清晰，没有"相关即因果"的谬误
□ 反向论点已被考虑和回应
□ 没有"循环论证"（用结论证明前提）

数据质量：
□ 核心数字有时间标注
□ 数据冲突已处理（非回避）
□ [DATA GAP] 已标记
□ 没有用模糊描述掩盖数据缺失

洞察质量：
□ 至少3个Insight-A级洞察
□ 没有"显而易见"的平庸结论
□ So What已回答
□ 没有无意义的修饰语（"非常""极其""显著"需有数据支撑）

行动导向：
□ 结论可以指导具体决策
□ 建议有优先级和执行条件
□ 风险点有监测指标
```
