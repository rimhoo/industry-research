# 质量闸门 Quality Gates
## 世界级研报标准定义 + 分层自检清单

> 本文件定义了"超越高盛/麦肯锡水准"的可操作标准。
> 每份报告完成后，必须通过本文件的质量检查。

---

## 一、世界级研报的五个维度基准

### 维度1：数据深度（Data Depth）

| 层次 | 普通研报 | 高盛/麦肯锡 | IRAM目标 |
|------|---------|------------|---------|
| 来源数量 | 3-5个来源 | 15-30个来源 | 20-40个来源 |
| 数据时效 | 常常2-3年前 | 通常12个月内 | 核心数据18个月内，有实时信号 |
| 数据层次 | 宏观+行业 | 宏观+行业+公司 | 宏观+行业+公司+另类数据 |
| 原创数据 | 几乎无 | 有一手调研 | 有AI辅助的信号挖掘 |
| 数据透明度 | 引用不规范 | 标注来源 | 完整来源链+置信度标注 |

**执行标准**：
```
□ 核心市场数据：≥3个权威来源交叉验证
□ 头部公司数据：来自年报/财报等一手来源
□ 趋势判断：有历史数据支撑，不是"专家认为"
□ 所有数字：来源+时间标注完整
□ [DATA GAP]：明确标注，不以模糊描述替代
```

---

### 维度2：分析深度（Analytical Depth）

| 层次 | 普通研报 | 高盛/麦肯锡 | IRAM目标 |
|------|---------|------------|---------|
| 分析框架 | 描述现象 | 有结构性框架 | 有原创框架+跨行业类比 |
| 因果分析 | 几乎无 | 部分有 | 每个结论有明确因果链 |
| 反驳考量 | 无 | 偶尔提及 | 系统性反驳+置信度 |
| 情景分析 | 无或流于形式 | 通常3个情景 | 2×2矩阵+概率+监测信号 |
| 非共识判断 | 跟随市场 | 有1-2个 | 每报告≥3个差异化判断 |

**执行标准**：
```
□ 五力分析：使用"穿透问题"升级版本（见synthesis-engine.md）
□ 利润池：量化各环节利润分配，有流动方向判断
□ 情景分析：≥2×2矩阵，有概率估算
□ 反共识：至少1个"我们与市场的核心分歧"章节
□ 因果链：结论段必须能回答"为什么"，不是"what"
```

---

### 维度3：洞察质量（Insight Quality）

世界顶级研报之所以被引用，是因为它们提供了**不可替代的认知**。

**洞察质量分级标准**：

```
★★★★★ Tier-1 洞察：
"独特数据 + 反直觉结论 + 可操作含义"
例（高盛2023）："美国科技公司用工成本每增加10%，
                 将使EPS下降约3.2%——但这个压力
                 在未来18个月会被AI提效抵消"

★★★★☆ Tier-2 洞察：
"交叉数据 + 结构性判断 + 明确含义"
例："中国新能源汽车渗透率已达38%，但二三线城市
    仅22%，这个差距意味着未来5年的增量市场
    几乎全来自低线城市——与一线城市打法完全不同"

★★★☆☆ Tier-3 洞察：
"有数据的结论"（达标下限）
例："X行业前三名市场份额合计65%，过去3年提升12pp"

★★☆☆☆ 描述性陈述（不可接受）：
"X行业竞争激烈，市场前景广阔，企业面临挑战"
```

**执行标准**：
```
□ 每份报告：Tier-1 ≥ 3个
□ 每份报告：Tier-1+2 ≥ 40% 的内容密度
□ 零容忍：消除所有 Tier描述性陈述
□ "So What"测试：每个洞察都能回答"所以呢？"
```

---

### 维度4：结构清晰度（Structural Clarity）

| 评估项 | 不合格 | 及格 | 优秀 |
|--------|--------|------|------|
| 标题设计 | 描述性标题 | 主题词标题 | 结论式标题 |
| 逻辑流 | 无明显逻辑 | 基本有序 | MECE + 相互强化 |
| 数据图表 | 无图或堆砌 | 有图有说明 | 图表即论据 |
| 摘要 | 有但是废话 | 有核心结论 | 独立阅读价值 |
| 可扫读性 | 必须通读 | 可粗读 | 30秒能获取核心信息 |

---

### 维度5：行动导向（Actionability）

```
世界级研报的终极标准：读完之后，决策者知道该做什么。

每份报告的结论部分必须包含：
□ 针对投资者的行动含义
□ 针对从业者的行动含义
□ 触发条件（什么信号出现时需要重新审视结论）
□ 监测指标清单（用于跟踪核心假设是否仍然成立）
```

## 二、分层自检清单

### Level A：必须通过（否则不可发布）

```
基础数据完整性：
□ 行业规模数据：有具体数字，有来源，有时间
□ 主要玩家：≥5家公司的具体数据（非泛泛描述）
□ 核心趋势：有数据支撑的量化趋势，非方向性描述

结论可验证性：
□ 每个核心结论都有至少2个独立来源支撑
□ 没有"专家认为"类表述（无名专家 = 无效来源）
□ 预测数字标注了预测机构和方法论

格式完整性：
□ 执行摘要（独立阅读可理解）
□ 数据来源清单
□ 数据截止时间标注
□ [DATA GAP]标记完整
```

### Level B：应该通过（影响报告质量评级）

```
洞察质量：
□ ≥3个Tier-1/Tier-2洞察
□ ≥1个反共识判断（我们与市场的分歧）
□ 情景分析（至少2个情景）
□ 每个核心结论回答了"So What"

分析深度：
□ 价值链分析：有利润率分布，不只是环节描述
□ 竞争分析：有战略群组判断，不只是玩家列表
□ 技术分析：有颠覆向量评估，不只是技术描述
□ 财务分析：有行业利润率基准，不只是个别公司数据

叙事质量：
□ 标题为结论式（不是描述式）
□ 每章首段是结论（不是铺垫）
□ 无"综上所述"类套话段落
□ 图表有独立的解读文字
```

### Level C：优秀标准（超越麦肯锡/高盛）

```
数据创新：
□ 使用了至少1种另类数据源（招聘/APP/搜索等）
□ 有原创的数据处理/可视化
□ 引入了跨行业类比（从其他行业的历史中获得洞察）

分析创新：
□ 有原创框架（不只是套用波特五力等标准框架）
□ 有明确的观点（不是"一方面...另一方面..."）
□ 反驳意见有系统性回应（不只是提及）
□ 时间尺度叠加（短/中/长期分别分析）

表达创新：
□ 标题串联起来构成完整叙事
□ 核心洞察可提炼为"一页纸"而不失内容
□ 有记忆点（一个会被引用的数字/框架/判断）
```

## 三、世界级研报对标参考

### 高盛研究的标志性特征

```
✦ 明确的投资结论在封面页（超配/中性/低配）
✦ 12个月目标价/目标倍数 + 概率加权
✦ Bull/Base/Bear三情景及各自概率
✦ "Our key debate" — 明确说我们与市场的分歧是什么
✦ 财务预测精确到季度
✦ 详细的估值推导过程（显示所有假设）

IRAM如何超越：
→ 增加实时另类数据维度
→ 增加中国本土市场视角的深度
→ 情景分析增加"触发信号"监测清单
```

### 麦肯锡MGI的标志性特征

```
✦ 原创数据（一手调研 + 专有数据库）
✦ 原创框架（往往被行业沿用多年）
✦ 政策建议维度（面向政府和社会）
✦ 全球视角 + 国家/地区深度
✦ 历史类比（从历史案例推导当前）

IRAM如何超越：
→ 增加执行颗粒度（不只是战略方向，到具体行动）
→ 原创框架结合AI时代特征
→ 更快的迭代速度（研报可以是动态更新的）
```

### 摩根士丹利的标志性特征

```
✦ 行业深度 — 分析师专注一个行业多年
✦ 基本面与宏观的联动分析
✦ 精确的历史回测（结论经历史验证）
✦ 高质量的图表（信息密度高）

IRAM如何超越：
→ 历史分析 + 实时信号的结合
→ 更强的可视化和信息层次
→ 明确区分"事实"和"判断"的透明度
```

## 四、常见质量缺陷与修复方案

| 缺陷类型 | 症状 | 修复方案 |
|---------|------|---------|
| **数据空洞** | "市场规模较大" | 执行M02深度搜索，或标注[DATA GAP] |
| **因果缺失** | "X行业增长，因为消费升级" | 追问：消费升级具体如何带动？有哪些数据证明？ |
| **竞争流于表面** | "主要有A、B、C三家公司" | 执行C02画像，量化每家的收入/份额/利润 |
| **结论套话** | "面临机遇与挑战" | 强制列举具体机遇（有规模量化）和具体挑战 |
| **预测无依据** | "未来将持续增长" | 引用具体机构预测 + 驱动因子分析 |
| **图表无解读** | 图下无文字 | 每张图必须有1-2句"这张图说明了什么" |
| **无行动含义** | 结论只描述状态 | 强制回答：对投资者/从业者意味着什么？ |
| **反驳缺失** | 只有正向论据 | 构建Anti-thesis章节，主动提出反驳 |
```

## 五、报告发布前最终检查

```
5分钟扫读测试：
□ 看封面：能理解这份报告的核心判断吗？
□ 看目录：标题能串联成完整故事吗？
□ 看摘要：能独立传递所有核心信息吗？
□ 看每章首句：能快速抓取章节结论吗？

核心数字核查：
□ 市场规模数字：来源？时间？
□ 增速数字：基于什么时间段？
□ 份额数字：什么口径？来自谁？
□ 预测数字：谁预测的？方法论？

逻辑一致性检查：
□ 执行摘要的结论是否与正文一致？
□ 情景分析的结论是否与基准判断协调？
□ 不同章节的数据是否相互矛盾？

读者测试（模拟目标读者提3个问题）：
□ "你凭什么这么说？" → 能回答吗？
□ "如果你错了呢？" → 有风险章节覆盖吗？
□ "我该怎么做？" → 行动建议明确吗？
```
