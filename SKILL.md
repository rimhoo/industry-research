## 第一部分：执行协议

### 1.1 接收任务后的强制前置步骤

```
STEP 0: 任务解析 (Task Decomposition)
─────────────────────────────────────
1. 识别行业边界：用户说的"XX行业"的精确范围是什么？
   → 上游/下游/周边是否纳入？中国市场/全球市场？
2. 识别研究目的：
   □ 投资决策  □ 战略规划  □ 竞争监测  □ 市场进入  
   □ 政策分析  □ 学术研究  □ 创业机会  □ 并购尽调
3. 识别时间维度：历史回溯/现状快照/前瞻预测？
4. 识别核心问题：用户最核心的3个问题是什么？
5. 选择执行流水线：→ 见 第三部分
```

### 1.2 搜索执行原则

1. **多轮迭代**：每个 ARU 至少执行 3 轮搜索，后续轮次基于前轮结果钻取
2. **来源分级**：优先级 P1(一手数据) > P2(权威机构) > P3(分析师) > P4(媒体)
3. **时效管理**：数据需标注采集时间，超过18个月的定量数据需注明并寻找更新版本
4. **矛盾处理**：不同来源数据冲突时，保留所有版本并注明差异，不擅自取舍
5. **空洞标记**：搜索后仍无法填充的数据点需用 `[DATA GAP]` 显式标记

### 1.3 推理质量标准

每个洞察必须满足**「SCAI 四检」**：
- **S**ource（来源）：有具体来源支撑
- **C**ausality（因果）：有明确因果链，非相关关系
- **A**ction（行动）：导向可操作的结论
- **I**nvalidation（反驳）：考虑了主要反驳意见

### M系列：市场结构 (Market Structure) ×10

| ARU | 名称 | 核心问题 | 主要输出 |
|-----|------|----------|----------|
| **M01** | 行业边界勘测 | 什么算这个行业？边界在哪里？ | 行业定义 + 上下游图谱 |
| **M02** | 市场规模测算 | TAM/SAM/SOM各多少？增速如何？ | 规模数字 + 测算方法说明 |
| **M03** | 市场分层解剖 | 按哪些维度分层？各层价值量？ | 分层矩阵 + 层间流动逻辑 |
| **M04** | 价值链解剖 | 价值在哪个环节创造？利润在哪里沉淀？ | 价值链图 + 利润率分布 |
| **M05** | 生命周期定位 | 行业处于萌芽/成长/成熟/衰退哪个阶段？ | 阶段判断 + 阶段特征证据 |
| **M06** | 市场集中度分析 | CR3/CR5/HHI是多少？寡头还是碎片？ | 集中度指标 + 变化趋势 |
| **M07** | 定价机制解析 | 行业如何定价？谁有定价权？ | 定价模型图 + 定价权归属 |
| **M08** | 渠道矩阵分析 | 产品如何到达终端用户？各渠道占比？ | 渠道地图 + 份额数据 |
| **M09** | 需求驱动因子 | 什么驱动需求增长/萎缩？ | 驱动因子树 + 弹性估算 |
| **M10** | 供给结构分析 | 供给侧的产能/产量/在建项目？ | 供给格局图 + 产能数据 |

## 第四部分：搜索策略概要

> 详细查询模板见 → `references/search-protocols.md`

### 4.1 查询构造的五个维度

每个ARU的搜索需要覆盖五个维度：

```
1. 数量维度 → "XX行业市场规模 2024 亿美元"
2. 结构维度 → "XX行业竞争格局 市场份额 CR3"  
3. 趋势维度 → "XX行业增速 CAGR 预测 2025-2030"
4. 案例维度 → "XX行业领军企业 战略 财报"
5. 信号维度 → "XX行业 最新动态 并购 融资 政策"
```

### 4.2 来源分级体系

```
P1 一手数据源（权重×1.5）
├── 上市公司年报/财报/投资者关系
├── 监管机构公告（证监会/工信部/FDA等）
├── 政府统计年鉴/普查数据
└── 学术期刊（一区）

P2 权威机构来源（权重×1.3）
├── 国际机构：IMF/WB/BIS/WTO
├── 行业协会：各行业协会白皮书
├── 咨询报告：McKinsey/BCG/Bain/Gartner/IDC
└── 评级机构：Moody's/S&P/惠誉

P3 分析师研究（权重×1.0）
├── 投行研报：高盛/摩根士丹利/摩根大通
├── 专业媒体：FT/Bloomberg/Reuters/WSJ
└── 专业数据库：Wind/Bloomberg/FactSet

P4 一般媒体（权重×0.7）
├── 财经媒体：36氪/虎嗅/钛媒体
├── 综合媒体：Forbes/Fortune/Economist
└── 行业媒体（垂直）
```

## 第五部分：输出质量标准

> 详细标准见 → `references/quality-gates.md`

### 5.1 世界级研报的核心特征（参照基准）

| 机构 | 标志性特征 | 我们要超越的维度 |
|------|-----------|----------------|
| **高盛研究** | 明确的投资结论 + 定量模型 + 情景分析 | 增加实时数据 + AI信号维度 |
| **麦肯锡MGI** | 深度结构分析 + 原创框架 + 政策建议 | 增加执行路径 + 操作颗粒度 |
| **摩根士丹利** | 行业深度 + 估值精度 + 前瞻判断 | 增加中国本土视角 + 新兴市场 |
| **BCG** | 战略框架创新 + 变革管理视角 | 增加数字化视角 + 速度 |
| **IDC/Gartner** | 技术分类精确 + 采用曲线 | 增加财务维度 + 因果分析 |

### 5.2 最低质量阈值（每份报告必须通过）

```
□ 市场规模：有具体数字 + 来源 + 测算方法说明
□ 关键玩家：至少5家公司的具体数据（非泛泛描述）
□ 数据时效：核心定量数据不超过18个月
□ 原创洞察：至少3个"非常识性"判断，有论据支撑
□ 矛盾处理：存在数据矛盾时已标注和解释
□ 数据空洞：[DATA GAP] 标记完整，不以模糊描述代替
□ 行动导向：结论部分明确"So What"
□ 反驳考量：主要结论已考虑反驳意见
```
